  <action>这个任务完成数据库、runtime 与后端命令层的统一 suppression 合同，但不做前端 copy 路由 sweep。1. 在 `CaptureRuntime` 中加入 suppression registry，条目结构固定为 `SuppressionEntry { content_hash: String, expires_at_ms: i64 }`，并提供 `register_suppression_key(content_hash: String, ttl_ms: i64)`。这个 registry 与 Task 1 的 `last_observed_hash` 使用同一个 SHA256 十六进制 `content_hash` 键合同。2. 在 `Database::init()` 创建唯一索引之前，必须先执行一个事务安全的旧数据去重迁移，例如 `merge_existing_content_hash_duplicates(tx: &mut Transaction<'_, Sqlite>)`。合并规则写死为：每个 `content_hash` 保留 `created_at` 最大的那一行，若时间相同则保留 `id` 字典序最大的那一行；合并后的 `copy_count = SUM(copy_count)`；`created_at = MAX(created_at)`；`source_app`、`content_subtype`、`metadata`、`app_bundle_id`、`file_path` 继承自 survivor 行，若 survivor 为 `NULL` 才允许从同组中最近的非空值补齐；删除其余 loser 行；只有事务提交成功后才执行 `CREATE UNIQUE INDEX IF NOT EXISTS idx_clipboard_entries_content_hash_unique ON clipboard_entries(content_hash)`。3. 新持久化 SQL 改成 `INSERT ... ON CONFLICT(content_hash) DO UPDATE SET copy_count = clipboard_entries.copy_count + 1, created_at = excluded.created_at, source_app = COALESCE(excluded.source_app, clipboard_entries.source_app), content_subtype = COALESCE(excluded.content_subtype, clipboard_entries.content_subtype), metadata = COALESCE(excluded.metadata, clipboard_entries.metadata), app_bundle_id = COALESCE(excluded.app_bundle_id, clipboard_entries.app_bundle_id), file_path = COALESCE(excluded.file_path, clipboard_entries.file_path)`，避免后续重复写入抹掉已有较完整的元数据。4. 在 `src-tauri/src/commands.rs` 与 `src-tauri/src/state.rs` 增加诸如 `register_suppression_for_text(&content, 1500)` 的后端方法：文本先按与 runtime 相同的 SHA256 十六进制规则计算为 `content_hash`，再调用 `register_suppression_key(content_hash, 1500)`，然后才允许系统剪贴板写入。5. 在 `src-tauri/src/capture_runtime_tests.rs` 中去掉 `test_capture_runtime_single_worker_and_suppression` 的 ignore，并新增 `test_capture_runtime_dedupe_migration_merges_existing_duplicates`，专门 seed 旧数据库重复行并验证迁移与唯一索引创建成功。</action>
